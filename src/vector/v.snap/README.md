## NAME

_v.snap_: Snap points from a vector point map onto the nearest location on the
features of a vector line map. Original categories and attributes are preserved.

## DESCRIPTION

*v.snap* snaps each point of an input vector point map onto the nearest location
along the features of a vector line map. The snapping target is the closest
point on any line segment, not merely the nearest vertex, so points are placed
exactly on the line geometry.

Category values and attribute columns from the input point map are preserved in
the output.

If **dmax** is specified, only points within that distance (in map units) of any
line feature are snapped. By default, points beyond **dmax** are excluded from
the output. With the **-k** flag these points are instead retained at their
original location and flagged in an added *snapped* column (1 for snapped, 0 for
original).

## NOTES

### 3D input

If the input point map is 3D and the **-z** flag is set, the original Z coordinate
of each point is preserved on the snapped output. Only the X and Y coordinates
are moved onto the line; Z is carried through unchanged. The snapped elevation
therefore reflects the elevation of the original observation, not the elevation
of the line feature at the snapped location.

### Layers

Vector maps in GRASS may carry multiple layers, each with its own attribute
table. The **layer** option selects which layer of the input point map to
process; only features and attributes belonging to that layer are transferred to
the output.

### Points without a category

Points that have no category value (*cat* = -1) in the selected layer are
excluded from the output with a warning. Assign categories with
[v.category](v.category.md) before running *v.snap* if these points are needed
in the output.

### Input without an attribute table

If the input point map has no attribute table connected at the selected layer,
the output will contain only the *cat* and *snapped* columns generated by
*v.snap*. No attribute join is performed in that case.

## EXAMPLES

### Snap GPS points onto a road network

```sh
v.snap input=gps_points lines=roads output=gps_on_roads
```

### Snap only points within 50 map units

Points farther than 50 units from any road are dropped from the output:

```sh
v.snap input=gps_points lines=roads output=gps_on_roads dmax=50
```

### Keep unsnapped points at their original location

All input points appear in the output; distant ones carry *snapped = 0*:

```sh
v.snap -k input=gps_points lines=roads output=gps_all dmax=50
```

The result can be filtered afterwards, for example:

```sh
v.extract input=gps_all where="snapped = 1" output=gps_only_snapped
```

### Snap using a non-default layer

```sh
v.snap input=observations layer=2 lines=rivers output=obs_on_rivers
```
